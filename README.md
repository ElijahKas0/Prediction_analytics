# Виртуальный датчик для контроля процесса обжига в печи

## Описание
В данном проекте реализован виртуальный датчик (soft sensor) для оценки текущей концентрации выходного продукта в промышленной печи.  
Модель использует минутную телеметрию для предсказания лабораторных измерений, компенсируя их задержку (10–15 минут) и нерегулярность.

---

## Структура проекта
- **Lab02.ipynb** — Jupyter Notebook с полным пайплайном решения  
- **requirements.txt** — зависимости проекта  
- **data_train.csv** — обучающая телеметрия (минутные измерения, 16 каналов)  
- **target_train.csv** — лабораторные замеры (нерегулярные, с задержкой)  
- **data_test_small.csv** — тестовая телеметрия  
- **target_test_small.csv** — тестовые лабораторные замеры  

---

## Запуск
```bash
pip install -r requirements.txt
jupyter notebook Lab02.ipynb
```

Далее выполнить все ячейки последовательно.  
Для тестирования используется блок синхронизации test-данных.

## Ход работы

### 1. Разведочный анализ (EDA)
- Проанализирована структура данных и временные интервалы
- Выявлены пропуски и разделение каналов на плотные и разреженные
- Исследована нерегулярность лабораторных измерений
- Обнаружены выбросы и нестабильные режимы процесса
- Проведен анализ корреляций и лагов

### 2. Синхронизация данных
Введено понятие:

process_time = lab_time - delay

- Телеметрия согласована с целевой переменной без утечки будущей информации
- Подобрана оптимальная задержка (delay)

### 3. Инжиниринг признаков
Созданы признаки, отражающие динамику процесса:

- Лаговые признаки (lag features)
- Скользящие статистики:
  - mean
  - std
  - min
  - max
- Производные и темпы изменения
- Признаки редких каналов:
  - последнее значение (ffill)
  - возраст значения (age)
- Календарные признаки (час, день недели)

### 4. Построение моделей
Использованы следующие модели:

- **HistGradientBoosting** — основная модель
- **Random Forest** — ансамблевая модель
- **ElasticNet** — линейный baseline

### 5. Оценка качества
Использованы метрики:

- MAE
- RMSE
- MAPE
- WAPE
- Directional Accuracy

**Дополнительно:**
- Анализ остатков:
  - нормальность
  - автокорреляция
  - гетероскедастичность
- Важность признаков 

## Результаты
Лучшая модель (градиентный бустинг):

- **MAE:** ~0.06–0.07
- **RMSE:** ~0.07–0.09
- **MAPE:** ~32–41%
- **Directional Accuracy:** ~0.49–0.54

## Интерпретация результатов
- Модель обеспечивает достаточную точность для промышленного применения
- Ошибка прогноза находится в допустимых пределах
- Направление изменения целевой переменной предсказывается корректно

**Ключевые наблюдения:**
- Процесс обладает инерционностью
- История (лаги) критически важна
- Волатильность влияет на качество продукта
- Редкие датчики информативны при корректной обработке

## Итог
Разработанный виртуальный датчик позволяет:

- Оценивать качество продукта в реальном времени
- Компенсировать задержку лабораторных измерений
- Повысить оперативность управления процессом
